MicroVM Isolation via Forkd

Instead of launching a completely fresh microVM for every execution, forkd boots the system once, pre-loading large dependencies such as Python and NumPy, and then snapshots the full virtual machine state.
Every subsequent execution creates a new KVM VM backed by a MAP_PRIVATE memory mapping of the snapshot. This ensures that the Linux kernel automatically handles copy-on-write page distribution. The speculative child process inherits the parent's address space at the page level, allowing the AI agent to "branch" its thought process in under 3 milliseconds per GiB of memory.
